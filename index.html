Latar Belakang

Apa yang rusak di arsitektur lama?

Sistem transaksi Bank X mengalami kegagalan kritis saat menghadapi beban puncak hingga 1 juta transaksi per jam.

💥

Crash saat peak load

Sistem tidak mampu bertahan saat lonjakan trafik terjadi — seperti momen lebaran atau akhir tahun — dan berakhir dengan total downtime.

🐢

Latensi p95 hingga 10 detik

Arsitektur sinkron tanpa caching menyebabkan antrian request menumpuk. Setiap transaksi harus menunggu proses database selesai sebelum merespons.

🚫

Tidak ada mekanisme proteksi

Tidak ada rate limiting, circuit breaker, atau backpressure. Satu komponen gagal langsung menjalar ke seluruh sistem secara cascading.

📈

Biaya server membengkak

Strategi scaling yang tidak efisien menyebabkan over-provisioning sumber daya cloud, meningkatkan biaya operasional tanpa peningkatan performa nyata.

Solusi

Resilient Edge Architecture

Sistem dibangun ulang dengan pendekatan asynchronous event-driven — setiap komponen dirancang untuk gagal secara terkontrol dan pulih sendiri.

API Layer

Go + Gin API Service

Menangani seluruh request masuk dengan rate limiting per-IP dan trace ID propagation end-to-end.

Async Queue

Kafka (3 partisi)

Transaksi write diproses asinkron — client mendapat HTTP 202 Accepted instan tanpa menunggu database.

Caching

Redis Cache-Aside

Cache hit rate 100% pada endpoint read-heavy. Write-through diupdate worker setiap transaksi sukses.

Database

PostgreSQL Master-Replica

Pemisahan jalur write dan read. PgBouncer connection pooling mendukung hingga 10.000 client concurrent.

Resilience

Circuit Breaker + Retry

gobreaker melindungi semua external call. Retry dengan exponential backoff 100ms→200ms→400ms untuk transient error.

Orchestration

Kubernetes HPA

Auto-scale dari 2 hingga 10 pod saat CPU >70% atau Memory >80%. Scale-in otomatis setelah beban turun.

Observability

Prometheus + Grafana

Dashboard real-time: RPS, p95 latency, error rate, circuit breaker state, cache hit rate, pgxpool stats.

Fallback

MongoDB Logger

Transaksi yang gagal permanen disimpan ke MongoDB sebagai audit trail tanpa menghentikan sistem.

Traceability

Trace ID End-to-End

UUID Trace ID konsisten dari API → Kafka header → Worker di seluruh structured log JSON zerolog.
